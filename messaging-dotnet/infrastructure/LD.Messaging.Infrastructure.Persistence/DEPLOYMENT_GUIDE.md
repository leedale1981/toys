# CQRS Postgres Persistence Layer - Deployment & Setup Guide

## Quick Start

### Prerequisites

- PostgreSQL 18
- .NET 11.0 SDK
- Docker (optional, for containerized PostgreSQL)

### 1. Database Setup

#### Option A: Docker (Recommended for Development)

```bash
# Start PostgreSQL container
docker run -d \
  --name postgres-stock-records \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=stock_records \
  -p 5432:5432 \
  postgres:18-alpine
```

#### Option B: Local PostgreSQL Installation

```bash
# On Windows with chocolatey
choco install postgresql

# On macOS
brew install postgresql

# Then create the database
createdb -U postgres stock_records
```

#### Option C: Docker Compose (Full Stack)

Add to your `docker-compose.yml`:

```yaml
version: '3.8'
services:
  postgres:
    image: postgres:18-alpine
    container_name: stock-records-db
    environment:
      POSTGRES_DB: stock_records
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres  # ⚠️ Change in production
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
```

Run with: `docker-compose -f docker-compose.yml up -d postgres`

### 2. Initialize Database

#### Option A: EF Core Migrations (Recommended)

```powershell
# Navigate to the infrastructure project
cd infrastructure\LD.Messaging.Infrastructure.Persistence

# Apply migrations
dotnet ef database update \
    --project . \
    --startup-project ..\..\ingestion-service\LD.Messaging.Ingestion

# Or from the ingestion service directory
cd ingestion-service\LD.Messaging.Ingestion
dotnet ef database update \
    --project ..\..\infrastructure\LD.Messaging.Infrastructure.Persistence \
    --startup-project .
```

#### Option B: Manual SQL Script

```bash
# Linux/macOS
psql -h localhost -U postgres -d stock_records -f infrastructure/LD.Messaging.Infrastructure.Persistence/setup-database.sql

# Or connect interactively
psql -h localhost -U postgres

# Then in psql:
\c stock_records
\i infrastructure/LD.Messaging.Infrastructure.Persistence/setup-database.sql
```

### 3. Configure Connection String

Edit `ingestion-service/LD.Messaging.Ingestion/appsettings.json`:

```json
{
  "ConnectionStrings": {
    "StockRecordsDb": "Host=localhost;Port=5432;Database=stock_records;Username=postgres;Password=postgres"
  }
}
```

### 4. Run the Ingestion Service

```powershell
cd ingestion-service\LD.Messaging.Ingestion
dotnet run
```

The service will:
1. Connect to PostgreSQL
2. Listen to Kafka topics (FTSE500, NYSE, NASDAQ)
3. For each message, create a `SaveStockRecordsCommand`
4. Execute the command handler to persist records
5. Log success with row count

## Architecture Details

### CQRS Pattern Implementation

```
┌─────────────────────────────────────────────────┐
│         Kafka Message                           │
│  (Ftse500Data, NyseData, NasdaqData)            │
└────────────────┬────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────────┐
│    MassTransit Consumer                         │
│  (Ftse500Consumer, NyseConsumer, etc.)          │
└────────────────┬────────────────────────────────┘
                 │ Creates
                 ▼
┌─────────────────────────────────────────────────┐
│  SaveStockRecordsCommand (CQRS Command)         │
│  Records: IReadOnlyList<StockRecord>            │
│  Exchange: string (FTSE500, NYSE, NASDAQ)       │
│  RecordDate, RecordTime, FileName               │
└────────────────┬────────────────────────────────┘
                 │ Passed to handler
                 ▼
┌─────────────────────────────────────────────────┐
│  SaveStockRecordsCommandHandler                 │
│  1. Validates command                           │
│  2. Maps domain records to DB entities          │
│  3. Begins transaction                          │
│  4. Executes parameterized INSERT               │
│  5. Commits or rolls back                       │
└────────────────┬────────────────────────────────┘
                 │ EF Core + Npgsql
                 │ (Parameterized queries)
                 ▼
┌─────────────────────────────────────────────────┐
│  PostgreSQL 18 Database                         │
│  INSERT INTO ingestion.stock_records            │
│  VALUES (@p0, @p1, @p2, ...)                    │
└─────────────────────────────────────────────────┘
```

### Security Model

**Threat**: SQL Injection via user input

**Defense Layers**:

1. **EF Core Parameterization**: All dynamic values are parameters
   ```sql
   -- Generated by EF Core:
   INSERT INTO ingestion.stock_records (...) VALUES (@p0, @p1, @p2, ...)
   ```

2. **Command Validation**: 
   - Exchange whitelist (FTSE500, NYSE, NASDAQ)
   - Symbol length check (max 10)
   - Price range validation
   - String trimming

3. **Database Constraints**:
   ```sql
   CHECK (exchange IN ('FTSE500', 'NYSE', 'NASDAQ'))
   CHECK (symbol <> '')
   CHECK (open >= 0) -- and other price checks
   ```

4. **Transactional Integrity**:
   - Atomic insert of all records
   - All-or-nothing semantics

## Database Schema

### Table: `ingestion.stock_records`

| Column | Type | Constraints | Purpose |
|--------|------|-----------|---------|
| id | SERIAL | PK | Auto-increment identifier |
| symbol | VARCHAR(10) | NOT NULL | Stock ticker (AAPL, MSFT, etc.) |
| name | VARCHAR(255) | NOT NULL | Company name |
| open | NUMERIC(19,4) | NOT NULL, >= 0 | Opening price |
| high | NUMERIC(19,4) | NOT NULL, >= 0 | Highest price |
| low | NUMERIC(19,4) | NOT NULL, >= 0 | Lowest price |
| close | NUMERIC(19,4) | NOT NULL, >= 0 | Closing price |
| volume | BIGINT | NOT NULL, >= 0 | Trading volume |
| change_percent | NUMERIC(5,2) | NOT NULL | Percentage change |
| exchange | VARCHAR(20) | NOT NULL, IN (...) | Market source |
| record_date | DATE | NOT NULL | Date recorded |
| record_time | TIME | NOT NULL | Time recorded |
| file_name | VARCHAR(255) | NOT NULL | Audit trail |
| created_at_utc | TIMESTAMP | NOT NULL, DEFAULT NOW() | Ingestion timestamp |

### Indexes (Performance)

```sql
-- Symbol lookup: O(log n) instead of O(n)
CREATE INDEX idx_stock_records_symbol ON ingestion.stock_records (symbol);

-- Exchange queries
CREATE INDEX idx_stock_records_exchange ON ingestion.stock_records (exchange);

-- Time-range queries
CREATE INDEX idx_stock_records_date_exchange ON ingestion.stock_records (record_date, exchange);

-- Audit/retention policies
CREATE INDEX idx_stock_records_created ON ingestion.stock_records (created_at_utc DESC);
```

### Verify Persistence

```powershell
# Connect to database
psql -h localhost -U postgres -d stock_records

# Count records
SELECT COUNT(*) FROM ingestion.stock_records;

# View latest records
SELECT symbol, name, close, created_at_utc 
FROM ingestion.stock_records 
ORDER BY created_at_utc DESC 
LIMIT 10;

# Detailed query
SELECT 
    symbol, name, exchange, record_date, 
    open, high, low, close, 
    change_percent, volume, 
    created_at_utc
FROM ingestion.stock_records
WHERE exchange = 'NYSE'
ORDER BY record_date DESC, record_time DESC
LIMIT 20;
```

## Troubleshooting

### Connection Refused

```
Error: "could not connect to server: Connection refused"
```

**Solution**: Ensure PostgreSQL is running
```bash
# Check service status
sudo systemctl status postgresql    # Linux
brew services list                  # macOS
# Or check Docker container
docker ps | grep postgres
```

### Authentication Failed

```
Error: "FATAL: password authentication failed"
```

**Solution**: Verify credentials in connection string
```json
"ConnectionStrings": {
  "StockRecordsDb": "Host=localhost;Port=5432;Database=stock_records;Username=postgres;Password=postgres"
}
```

### Table Doesn't Exist

```
Error: "relation \"ingestion.stock_records\" does not exist"
```

**Solution**: Run migrations
```powershell
dotnet ef database update --project infrastructure/LD.Messaging.Infrastructure.Persistence
```

### Migration Conflicts

```
Error: "The migration '...' has already been applied to the database"
```

**Solution**: Check migration history
```sql
SELECT * FROM __EFMigrationsHistory;

-- Remove problematic migrations if needed (carefully!)
DELETE FROM __EFMigrationsHistory WHERE MigrationId = '...';
```

## Performance Tuning

### Connection Pooling

Npgsql automatically manages connection pooling (default: min 1, max 25):

```csharp
// Customize in DbContext options:
options.UseNpgsql(connectionString, npgsqlOptions =>
{
    npgsqlOptions.UseAdminConnection(true); // For admin tasks
});
```

### Batch Operations

The command handler inserts all records in a single batch:

```csharp
// All 100 records in one transaction
await _dbContext.StockRecords.AddRangeAsync(entities);
await _dbContext.SaveChangesAsync();
```

### Query Performance

Use indexes for common queries:

```sql
-- Fast exchange lookup
SELECT * FROM ingestion.stock_records 
WHERE exchange = 'NYSE' 
ORDER BY record_date DESC;

-- Fast time-range query
SELECT * FROM ingestion.stock_records 
WHERE record_date BETWEEN '2026-05-01' AND '2026-05-17'
  AND exchange = 'FTSE500';
```

## Backup & Recovery

### Backup Database

```bash
# Full backup
pg_dump -h localhost -U postgres stock_records > backup.sql

# Compressed backup
pg_dump -h localhost -U postgres -Fc stock_records > backup.dump
```

### Restore Database

```bash
# From SQL file
psql -h localhost -U postgres stock_records < backup.sql

# From compressed file
pg_restore -h localhost -U postgres -d stock_records backup.dump
```
